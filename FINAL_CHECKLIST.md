# چک‌لیست نهایی (Final Checklist)

این چک‌لیست برای اطمینان از پوشش تمام نیازمندی‌ها و جلوگیری از فراموشی موارد مهم (Anti-Forgetting) تهیه شده است.

## ۱. موارد مربوط به کد و عملکرد

- [x] **ریپازیتوری:** پروژه با موفقیت کلون شده و ساختار آن تحلیل شده است.
- [x] **فارسی‌سازی:** تمام متون اصلی در جریان‌های کاربر به فارسی ترجمه شده‌اند.
- [x] **راست‌چین (RTL):** استایل‌های اصلی برای پشتیبانی از چیدمان راست‌چین اعمال شده‌اند.
- [x] **واحد پول:** واحد پول به «تومان» تغییر یافته و اعشار از نمایش قیمت‌ها حذف شده است.
- [x] **تاریخ شمسی:** تاریخ‌ها در بخش‌های اصلی (سفارشات، گزارش‌ها) به شمسی نمایش داده می‌شوند.
- [x] **نقشه:** سرویس نقشه گوگل با «نشان» جایگزین شده است.
- [x] **مناطق سرویس:** قابلیت تعریف مناطق بر اساس چندضلعی (Polygon) در پنل ادمین پیاده‌سازی شده است.
- [x] **درگاه پرداخت:** درگاه زرین‌پال (با حالت Sandbox) پیاده‌سازی شده است.
- [x] **ظرفیت روزانه:** قابلیت `stock` برای مدیریت موجودی روزانه غذاها اضافه شده است.
- [ ] **منطق کاهش موجودی:** **(نیاز به بررسی در فاز تست)** آیا پس از ثبت هر سفارش، از `stock` محصول کم می‌شود؟
- [ ] **پیکربندی محیطی (`.env`):** آیا کلیدهای API (نشان، زرین‌پال) و تنظیمات درگاه پرداخت (Sandbox/Production) از طریق فایل‌های پیکربندی قابل مدیریت هستند؟ (فرض: بله، اما نیاز به تایید نهایی دارد).

## ۲. موارد مربوط به تجربه کاربری (UX)

- [ ] **لحن ترجمه‌ها:** آیا لحن ترجمه‌ها برای مخاطب هدف (مامان‌ها، مشتریان) مناسب است؟
- [ ] **ساده‌سازی پنل مامان‌ها:** آیا منوهای غیرضروری در پنل مامان‌ها مخفی یا حذف شده‌اند تا از سردرگمی جلوگیری شود؟ (پیشنهاد شده در `ANALYSIS_MOMS_PANEL.md`)
- [ ] **پیام‌های خطا:** آیا پیام‌های خطا (مانند خطای پرداخت، آدرس خارج از محدوده) به فارسی روان و واضح نمایش داده می‌شوند؟
- [ ] **ایمیل‌ها و نوتیفیکیشن‌ها:** آیا قالب ایمیل‌ها و نوتیفیکیشن‌های ارسالی به کاربر (تایید سفارش، ارسال) فارسی‌سازی شده‌اند؟ (نیاز به بررسی).

## ۳. موارد فراموش‌شده احتمالی (Anti-Forgetting)

- [ ] **مدیریت کاربران (ادمین):** آیا پنل ادمین برای مدیریت مامان‌ها و مشتریان به درستی کار می‌کند؟ (فیلتر، جستجو، ویرایش).
- [ ] **صفحات استاتیک:** آیا صفحاتی مانند «درباره ما» و «تماس با ما» بررسی و در صورت نیاز، محتوای آن‌ها بومی‌سازی شده است؟
- [ ] **وابستگی‌های خارجی:** آیا تمام وابستگی‌ها به سرویس‌های خارجی که در ایران مسدود هستند (مانند فونت‌های گوگل) شناسایی و با جایگزین‌های مناسب تعویض شده‌اند؟ (نیاز به بررسی).
- [ ] **لاگ‌ها و مانیتورینگ:** آیا سیستم لاگ مناسبی برای خطاهای پرداخت یا ارتباط با سرویس‌های خارجی در نظر گرفته شده است؟ (پیشنهاد شده برای درگاه پرداخت).
- [ ] **امنیت:** آیا موارد امنیتی اولیه مانند جلوگیری از SQL Injection و XSS (که فریم‌ورک Yii به صورت پیش‌فرض مدیریت می‌کند) بررسی شده‌اند؟

## ۴. آماده‌سازی برای تحویل

- [x] **مستندات:** فایل‌های `ANALYSIS.md`, `ROADMAP.md`, `TEST_SCENARIOS.md` و `FINAL_CHECKLIST.md` ایجاد شده‌اند.
- [ ] **اسکریپت دیتابیس:** آیا فایل `karenderia.sql` شامل تمام تغییرات لازم برای راه‌اندازی اولیه سیستم است؟ (بله، تغییرات اعمال شده است).
- [ ] **راهنمای نصب و راه‌اندازی:** آیا یک راهنمای کوتاه برای نصب وابستگی‌ها و اجرای پروژه (Backend و Frontend) تهیه شده است؟ (پیشنهاد می‌شود یک `README.md`简简 برای این منظور ایجاد شود).
